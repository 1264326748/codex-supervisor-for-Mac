# Contributing

感谢你愿意参与这个项目。

## 提交前检查

请先在本地通过：

```bash
pnpm test
pnpm build
```

## 开发建议

1. 优先提交小而清晰的 PR（一个 PR 解决一个问题）；
2. 变更行为时请同步补测试；
3. 如果涉及运行时策略改动，请在 PR 描述里写清楚：
   - 变更前行为
   - 变更后行为
   - 风险点与回滚方式

## 代码风格

- 保持现有目录结构与命名习惯；
- 不引入与当前栈不一致的重型依赖；
- 以可维护和可追踪日志为优先。

## Issue 模板建议

提 issue 时建议包含：

1. 复现步骤
2. 预期行为
3. 实际行为
4. 关键日志（`data/logs/*.ndjson` 的相关片段）
5. 运行环境（macOS 版本、Node 版本、tmux 版本）
