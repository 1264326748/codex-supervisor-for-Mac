# Security Policy

## 支持范围

当前默认仅对 `main` 分支上的最新代码提供安全修复支持。

## 报告方式

如果你发现潜在安全问题，请不要先公开披露。建议通过仓库 Issue 私下描述，并至少包含：

1. 问题类型与影响范围；
2. 复现步骤；
3. 影响版本/分支；
4. 可选的修复建议。

## 处理流程

维护者收到报告后会：

1. 先确认可复现性与影响等级；
2. 在必要时先提交修复，再公开说明；
3. 修复完成后在变更记录中注明安全相关修复点。

## 注意事项

- 请不要在公开 issue 中直接贴敏感凭据或可利用细节；
- 如果问题涉及第三方依赖，请同时附上依赖版本信息。
